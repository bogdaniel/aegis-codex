# Code Review Checklist

- [ ] SOLID and Clean Architecture boundaries respected
- [ ] Public contracts typed and documented
- [ ] Input validation at boundaries; no unsanitized inputs
- [ ] No secrets or sensitive data in code/logs
- [ ] Parameterized queries/ORM; migrations reviewed
- [ ] Error handling with actionable messages; no leaks
- [ ] Structured logging with appropriate levels
- [ ] Efficient algorithms and data structures
- [ ] Deterministic tests with sufficient coverage
- [ ] Performance considerations for hot paths
- [ ] API versioning and backward compatibility
- [ ] Docs and ADRs updated
